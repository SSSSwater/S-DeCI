# S-DeCI 模块3/4改进实验记录（2026-07-14）

## 目标

继续改进模块3/4，但不通过关闭模块、降低模块权重或堆叠额外损失来换指标。本轮重点放在模块3的图级双曲读出，使 `z_global` 的类别方向更稳定，再交给模块4 Busemann-HPEC 做方向证据分类。

## 参考依据

- Hyperbolic GNN / H2H-GCN 一类方法强调，双曲图聚合如果全部拉回原点切空间做均值，会在节点远离原点时产生几何失真。
- Einstein midpoint 是 Poincare Ball 中常用的低复杂度几何中心近似：先把 Poincare 点映到 Klein 坐标，用 Lorentz gamma 加权求中心，再映回 Poincare Ball。
- 这与当前 S-DeCI 叙事一致：模块3负责在双曲空间表达脑网络层级结构，模块4用 Busemann 方向证据判断类别。

## 新增实现

新增参数：

```text
--hgcn-einstein-readout-weight
```

默认值为 `0.0`，因此不改变当前最佳默认路线。

实现位置：

- `layers/hyperbolic_gcn_layer.py`
- `models/S_DeCI.py`
- `test_mdd_best_config.py`
- `run_cv.py` 默认字典

计算方式：

1. 模块3得到每个 ROI 的 Poincare 节点表示 `x_i`。
2. 将 `x_i` 转为 Klein 坐标：

```latex
k_i = \frac{2x_i}{1 + \|x_i\|^2}
```

3. 计算 Lorentz gamma：

```latex
\gamma_i = \frac{1}{\sqrt{1-\|k_i\|^2}}
```

4. 用节点权重 `w_i` 计算 Einstein midpoint：

```latex
k_{\mathrm{mid}} =
\frac{\sum_i w_i \gamma_i k_i}{\sum_i w_i \gamma_i}
```

5. 映回 Poincare Ball，并取 `logmap0` 得到切空间方向 `v_{\mathrm{ein}}`。
6. 与原有 `mean/std` 图级读出融合：

```latex
z_{\mathrm{tan}} =
(1-\alpha) z_{\mathrm{tan}} + \alpha v_{\mathrm{ein}}
```

其中 `\alpha = hgcn_einstein_readout_weight`。

## 当前判断

- Einstein midpoint readout 的理论方向合理，且没有增加额外损失或分类分支。
- 但短测只带来很小的 Macro-F1 改善，模块4分支测试泛化仍未明显增强，因此暂不改默认，也不做完整5fold。
- 保留该参数作为后续模块3几何读出的消融开关。当前默认仍保持 `hgcn_einstein_readout_weight=0.0`。

## 下一步建议

后续更值得尝试的是减少模块3 readout 中 `z_global` 的训练集过拟合，而不是继续改模块4后处理。一个可能方向是让模块3读出基于稳定的因果子网络或低维脑网络统计生成 `z_global`，但需要先用短测确认模块4分支在测试集上的 AUC 和 Macro-F1 是否同步提升。

## HPEC 原型 EMA 延迟更新筛查

### 动机

模块4当前使用多原型和 Sinkhorn EMA 更新。小样本场景下，如果训练早期 `z_global` 还没稳定，原型可能被早期 batch 的方向噪声拖动，导致测试集 HPEC 分支多数类化。因此新增一个很小的调度参数：

```text
--hpec-ema-start-epoch
```

默认值为 `0`，保持原行为。设置为大于 0 时，前几轮不更新 HPEC prototype EMA，让模块3先形成较稳定的 `z_global`，之后再进行 Sinkhorn EMA。

## 因果加权读出与 Busemann 边界校准复测

### 动机

当前完整模型中，模块3/4分支在训练集上可以得到很高 AUC，但测试集上常退化为多数类预测。这说明问题不是 Busemann energy 完全无效，而是双曲全局点 `z_global` 的测试分布不够稳定，或者 Busemann horosphere 边界在 raw threshold 下校准不足。本轮只测试轻量改动，不新增复杂损失：

1. `causal_weighted_mean_std`：模块3读出时用模块2/分类图的节点权重参与 mean/std，使 `z_global` 更关注因果图中重要 ROI。
2. `hpec_busemann_class_bias_weight`：给每类 Busemann horosphere 一个很小的可学习偏置，用于校准类别边界位置。
3. `hpec_input_radius_min/max`：只限制送入 HPEC 的半径范围，减少半径漂移对 Busemann 方向证据的干扰。

## HPEC 原型学习率缩放完整复测

### 动机

当前最佳配置说明“可训练原型”是有收益的，但模块3/4分支仍然存在训练集贴合较快、测试集不稳定的问题。因此本轮只新增一个很小的优化器级别调度参数：

```text
hpec_prototype_lr_scale
```

它不改变模型结构，也不新增损失项，只把 HPEC 原型和 Busemann class bias 单独放入一个 optimizer parameter group。当该值小于 1 时，原型方向更新更慢，目的是避免 Busemann 原型过快追随训练集局部分布。

### 当前判断

- 慢速原型更新的动机合理，但完整 5fold 下没有超过当前基线，Acc、Macro-F1 和 AUC 都略低。
- 该结果说明当前瓶颈不只是“原型更新太快”，而更可能来自模块3读出的 `z_global` 测试分布与模块4 Busemann 边界之间的泛化不稳定。
- 因此 `hpec_prototype_lr_scale` 保留为消融参数，默认仍为 `1.0`。
- 后续模块3/4若继续改进，应优先从“让双曲图级表示本身更有类别稳定性”入手，而不是继续放慢或固定原型。

## 模块3读出候选复测

### 参考依据

本轮优先考虑不新增损失项的模块3读出改动。依据是：

- HGCN 一类方法通常在切空间中做聚合，以避免直接在双曲流形上求复杂 Fréchet mean。
- Einstein midpoint 提供 Poincare Ball 上较稳定的闭式双曲均值近似，适合作为图级中心点的轻量几何校正。
- 图分类和脑网络分析都强调层级/子网络结构；因此“因果子网络读出”比单纯节点级 attention 更符合脑区到功能网络再到全脑的叙事。

### 代码核查

核查 `layers/hyperbolic_gcn_layer.py` 后确认：当前默认 `hgcn_readout_mode=mean_std` 已经只拼接 `mean + std`，没有再拼接 `max_pool`。`max_pool` 只在其他 legacy/非默认 readout 模式中保留。因此之前“去掉切空间 max pooling”的建议已经落实在默认主线中，不需要重复修改。

### 短测结果

MDD / AAL116 / 1fold / 25epoch / 模块1-4全启用：

| 配置 | Acc | Precision | Recall | Macro-F1 | AUC | 判断 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| `hgcn_readout_mode=causal_subnetwork`, `count=4`, `topk=12`, `weight=0.5` | 68.75% | 64.18% | 59.15% | 59.01% | 58.70% | 硬选择因果子网络导致全局信息损失，不进入完整5fold |
| `hgcn_readout_mode=mean_std`, `hgcn_graph_readout_alpha=0.15` | 71.25% | 68.51% | 61.95% | 62.29% | 60.52% | 与同折默认基本持平，未显示明显收益 |
| `hgcn_readout_mode=mean_std`, `hgcn_graph_readout_alpha=0.30` | 71.25% | 68.51% | 61.95% | 62.29% | 60.66% | 与同折默认基本持平，未显示明显收益 |
| `hgcn_readout_mode=mean_std`, `hgcn_einstein_readout_weight=0.10` | 71.25% | 68.51% | 61.95% | 62.29% | 60.59% | 几何动机合理，但短测收益不足 |
| `hgcn_readout_mode=mean_std`, `hgcn_einstein_readout_weight=0.20` | 71.25% | 68.51% | 61.95% | 62.29% | 60.59% | 与 0.10 类似，且耗时略高 |

### 当前判断

- `causal_subnetwork` 的故事性很好，但当前实现的“硬选种子 + top-k ROI”会损失全脑稳定信息，短测直接低于默认，不应作为下一步主线。
- `graph_readout_alpha` 和 `einstein_readout_weight` 都是温和改动，短测没有破坏结果，但也没有明显超过默认，因此不值得直接跑完整 5fold。
- 当前模块3默认 `mean_std` 是稳定的，继续改 readout 的边际收益很低。模块34的主要瓶颈仍是：HPEC/双曲分支训练集可分，但测试集证据校准和泛化不足。
- 下一步更值得尝试的是模块4侧的“证据校准”而不是继续改模块3读出：例如只保留 Busemann energy 主证据，降低重复 prototype similarity 叠加，或使用不依赖测试 batch 统计的固定温度 margin 校准。但这些必须通过完整 5fold 证明，不能只靠短测判断。

## 模块4 Busemann 证据校准复测

### 动机

当前模块4默认会把 Busemann energy 作为主证据，同时再叠加同源的 `prototype_similarity` 证据。历史完整 5fold 显示这种叠加在最终 Acc 上最好，但从模型纯粹性看，它可能带来“重复使用同一 Busemann 方向证据”的疑问。因此本轮只测试已有轻量开关：

1. `hpec_avoid_busemann_double_count=1`：在 `energy_primary + busemann` 下只保留 energy 主证据，不再额外叠加 prototype similarity。
2. `hpec_residual_calibration=tanh_margin`：不用测试 batch 的均值/方差做校准，而是用固定有界 margin，减少测试 batch 统计波动。
3. 二者组合。

### 短测结果

MDD / AAL116 / 1fold / 25epoch / 模块1-4全启用：

| 配置 | Acc | Precision | Recall | Macro-F1 | AUC | 判断 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| `hpec_avoid_busemann_double_count=1` | 71.25% | 68.51% | 61.95% | 62.29% | 59.96% | 最终 Acc/F1 与同折默认相近，但 AUC 更低，不进入完整5fold |
| `hpec_residual_calibration=tanh_margin` | 70.00% | 68.54% | 58.28% | 56.99% | 58.77% | 固定 margin 校准削弱最终分类，不采用 |
| `hpec_avoid_busemann_double_count=1 + tanh_margin` | 70.00% | 68.54% | 58.28% | 56.99% | 58.84% | 组合后仍下降，不采用 |

### 当前判断

- 去掉重复 Busemann evidence 从“模型解释”角度更干净，但短测没有提升，且 AUC 下降。结合历史完整 5fold 中默认叠加更好，当前不改默认。
- `tanh_margin` 的固定校准没有解决测试集稳定性，反而让最终输出更接近 GCN fallback，模块4自身分支仍多数类化。
- 这说明模块4瓶颈不只是输出校准，而是 Busemann 分支从 `z_global` 中得到的类别证据本身仍不够稳定。
- 后续若继续改模块4，应优先考虑“让 Busemann 原型接收更稳的类方向输入”，而不是继续调后处理校准或关掉证据项。

## Busemann 固定打分半径复测

### 动机

Busemann ideal prototype 的核心是判断样本点朝向哪个双曲边界方向。理论上，方向对应类别证据，半径更接近层级深度或置信度。小样本训练中，如果半径过早过拟合训练集，则可以尝试在 Busemann 打分前把样本投到固定切空间半径，只保留方向证据：

```text
hpec_busemann_point_radius > 0
```

这个改动不新增模块、不新增损失，只改变 Busemann score 的输入点。默认 `0` 表示保留原始 `z_global` 半径。

### 短测结果

MDD / AAL116 / 1fold / 25epoch / 模块1-4全启用：

| 配置 | Acc | Precision | Recall | Macro-F1 | AUC | 判断 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| `hpec_busemann_point_radius=0.3` | 70.00% | 66.59% | 60.10% | 60.00% | 61.64% | AUC 略高，但 Acc/F1 低于同折默认，不进入完整5fold |
| `hpec_busemann_point_radius=0.5` | 68.75% | 64.46% | 58.25% | 57.62% | 61.77% | AUC 略高但分类明显下降，不采用 |

### 当前判断

- 固定半径让 Busemann 更“只看方向”，但最终 Acc/Macro-F1 下降，说明当前任务中半径信息仍有用。
- 这意味着 `z_global` 的半径不只是噪声，也可能携带模块3对样本层级深度/置信度的表达；强行固定会削弱模块4的判别。
- 因此 `hpec_busemann_point_radius` 继续保留为消融参数，默认仍为 `0`。
- 下一步不应继续把 Busemann 简化成纯方向分类，而应考虑让模块4更好地同时利用“方向 + 半径”：例如用低自由度的半径门控调节 Busemann evidence，而不是把半径直接去掉。

## Busemann 半径门控短测

### 动机

固定打分半径说明“只看方向”会降低 Acc/Macro-F1，但完全不处理半径又可能让训练集半径置信度过拟合。于是实现一个默认关闭的低自由度半径门控：

```text
hpec_busemann_radius_gate_weight
hpec_busemann_radius_gate_center
```

在 Busemann score 计算后，用样本的 Poincare 半径轻量调节方向证据：

```latex
g(z) = \mathrm{clip}\left(1 + \alpha(\|z\|_2 - r_0), 0.5, 1.5\right)
```

```latex
s'(z, p) = g(z) \cdot s_{\mathrm{Busemann}}(z, p)
```

其中 `alpha = hpec_busemann_radius_gate_weight`，`r0 = hpec_busemann_radius_gate_center`。默认 `alpha=0`，完全保持旧行为。

### 实现位置

- `layers/hpec_energy_layer.py`：在 Busemann score 后加入半径门控。
- `models/S_DeCI.py`：构造 HPEC 模块时传入门控参数。
- `test_mdd_best_config.py` / `run_cv.py`：增加命令行参数，默认关闭。

### 短测结果

MDD / AAL116 / 1fold / 25epoch / 模块1-4全启用：

| 配置 | Acc | Precision | Recall | Macro-F1 | AUC | 判断 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| `hpec_busemann_radius_gate_weight=0.25`, `center=0.3` | 71.25% | 68.51% | 61.95% | 62.29% | 60.45% | 与默认基本持平，未显示增益 |
| `hpec_busemann_radius_gate_weight=0.50`, `center=0.3` | 71.25% | 68.51% | 61.95% | 62.29% | 60.59% | 与默认基本持平，未显示增益 |

### 当前判断

- 半径门控是低复杂度、可解释的改动，但短测没有明显超过默认，不进入完整 5fold。
- 该实现保留为消融窗口，默认 `hpec_busemann_radius_gate_weight=0.0`，不影响当前最佳配置。
- 这一组进一步说明：模块4的问题不是单个半径缩放公式能解决，真正瓶颈仍在 `z_global` 类别方向和最终双视角融合的稳定性。
