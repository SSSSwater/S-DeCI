# 环境配置与运行说明

本文档用于说明本项目推荐的 Python 环境、PyTorch/CUDA 安装方式和基础验证步骤。由于 PyTorch 的 CUDA wheel 与本机显卡驱动、CUDA 运行时强相关，项目不在 `requirements.txt` 中固定 `torch==某个精确版本`，而是建议先按设备安装合适的 PyTorch，再安装其余依赖。

## 1. 推荐环境

- Python：建议使用 `3.10.x`，当前主要验证环境为 `Python 3.10.8`。
- 操作系统：Windows 优先；Linux 也可使用同样的 Python 依赖逻辑。
- PyTorch：建议 `torch>=2.4,<3.0`。当前本机验证过 `torch 2.11.0+cu128`。
- 数据目录：预处理后的数据集放在项目根目录 `dataset/` 下。
- 虚拟环境目录：建议使用项目根目录 `.venv/`。

## 2. 创建并激活虚拟环境

Windows PowerShell：

```powershell
py -3.10 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
```

Linux / macOS：

```bash
python3.10 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
```

## 3. 安装 PyTorch

请优先到 PyTorch 官方安装页根据自己的系统、CUDA 版本和安装方式生成命令：

```text
https://pytorch.org/get-started/locally/
```

常见情况如下。

### CUDA 12.8 或相近环境

如果机器支持 CUDA 12.8，可使用类似命令：

```powershell
python -m pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu128
```

### CUDA 12.1 环境

如果机器使用 CUDA 12.1，可使用类似命令：

```powershell
python -m pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

### 仅 CPU 环境

如果没有可用 GPU，或只想进行 CPU 冒烟测试：

```powershell
python -m pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
```

说明：不要直接把某台机器上的 `torch==版本+cuXXX` 写进通用 `requirements.txt`，否则其他 CUDA 版本的设备很容易安装失败或被错误降级。

## 4. 安装项目依赖

完成 PyTorch 安装后，再安装项目依赖：

```powershell
python -m pip install -r requirements.txt
```

`requirements.txt` 中的 `torch>=2.4,<3.0` 会检查 PyTorch 主版本范围。如果前一步已经安装了合适的 CUDA 版 PyTorch，通常不会重新安装或降级。

## 5. 验证环境

检查依赖冲突：

```powershell
python -m pip check
```

检查 PyTorch 与 CUDA：

```powershell
python -c "import torch; print('torch:', torch.__version__); print('cuda_available:', torch.cuda.is_available()); print('cuda_runtime:', torch.version.cuda); print('gpu_count:', torch.cuda.device_count()); print('gpu_name:', torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'CPU only')"
```

检查核心依赖是否可导入：

```powershell
python -c "import einops, geoopt, matplotlib, numpy, pandas, scipy, sklearn, torch, tqdm; from torch.utils.tensorboard import SummaryWriter; print('core imports ok')"
```

## 6. 冒烟训练

如果只想确认训练链路能跑通，可先使用较小配置：

```powershell
python test_training_smoke.py
```

如果要测试当前 S-DeCI 的数据集入口，可使用对应脚本，例如：

```powershell
python test_mdd_best_config.py --max-folds 1 --train-epochs 2 --visualize-causal 0
```

## 7. TensorBoard

训练过程默认会写入 TensorBoard scalar 日志。启动查看服务：

```powershell
.\start_tensorboard.ps1
```

或手动启动：

```powershell
tensorboard --logdir outputs\tensorboard
```

浏览器打开 `http://127.0.0.1:6006` 查看 loss、指标和模块诊断量趋势。

## 8. 常见问题

### 安装依赖时 PyTorch 被降级

先确认当前 PyTorch 是否满足版本范围：

```powershell
python -c "import torch; print(torch.__version__)"
```

若已经是 `2.4` 到 `3.0` 之间，但仍发生降级，通常是 pip 解析缓存或额外命令造成。建议重新按第 3 步安装本机 CUDA 对应的 PyTorch。

### CUDA 不可用

先确认显卡驱动和 PyTorch CUDA wheel 是否匹配：

```powershell
nvidia-smi
python -c "import torch; print(torch.version.cuda, torch.cuda.is_available())"
```

如果 `torch.cuda.is_available()` 为 `False`，训练仍可在 CPU 上运行，但速度会明显变慢。

### `python-docx` 缺失

生成周报等 Word 文档功能依赖 `python-docx`。执行以下命令即可补齐：

```powershell
python -m pip install -r requirements.txt
```
