# S-DeCI 模块 3/4 分类边界实验记录（2026-07-17）

## 1. 实验目的

本轮实验检查以下问题：

1. 模块 3 的全脑 `mean_std` 读出是否丢失局部因果角色差异。
2. 模块 4 的 batch 级可靠 TP EMA 是否导致多 prototype 聚拢。
3. Busemann energy 与 prototype similarity 重复使用同一证据是否影响泛化。
4. 更分散的 prototype 是否能直接带来更好的测试分类边界。

## 2. 文献依据

- HGCN：双曲空间适合表示图中的潜在层级结构，但图传播和读出需要保持几何一致性。
- Lorentzian GCN：使用流形质心比原点切空间直接平均具有更明确的几何意义。
- Hyperbolic Entailment Cones：cone 主要描述偏序包含关系，普通疾病二分类不能仅依赖 cone 形成边界。
- BrainCL：使用多阶拓扑、显著性互补视图以及可靠 TP EMA prototype 提高脑网络表征判别性。
- Prototypical Networks：prototype 应当对应所用度量空间内的类别中心。

## 3. 实现的消融开关

### 3.1 因果角色双曲读出

根据有向图的入度和出度构造四种无标签角色：

- `global`：全脑均匀权重。
- `source`：出度高于入度的因果源 ROI。
- `sink`：入度高于出度的因果汇 ROI。
- `hub`：总因果连接强度较高的枢纽 ROI。

每个角色使用 Einstein midpoint 得到一个 Poincare Ball 中的双曲中心。模块 4 为四个角色分别维护 prototype，并使用每类 4 个低自由度门控权重聚合角色 energy。

相关参数：

- `use_causal_role_readout`
- `causal_role_temperature`
- `hpec_causal_role_energy_weight`

### 3.2 epoch 级流形 prototype EMA

新增 `epoch_reliable_frechet_ema`：

1. 每个训练 batch 只缓存 detached 表征，不立即移动 prototype。
2. epoch 结束后按真实类别、置信度和互补视图一致性生成连续可靠度。
3. 使用 soft assignment 避免某个 prototype 长期收不到样本。
4. 使用 Karcher 迭代计算 Poincare Ball 中的加权 Frechet mean。
5. 沿测地线执行慢速 EMA，整个过程位于 `torch.no_grad()` 中。

该路径能够将 prototype 最大余弦相似度从约 `0.86` 降到 `0.56`，角色版本进一步降到约 `0.39`；prototype 占用熵约为 `0.998`，不存在明显饥饿。但这些几何改善没有稳定转化为分类提升。

## 4. 完整 MDD 结果

数据集为 MDD，图谱为 AAL116，序列长度为 230，训练为 5 fold / 50 epoch，指标取每折最后 epoch 的原始 `0.5` 阈值测试结果。

## 5. 结论

1. 因果角色读出在部分单折实验中提高了 AUC，但完整五折没有形成稳定收益。
2. epoch Frechet EMA 能稳定分散 prototype，却没有改善同一折最终分类指标。
3. 角色路径耗时约为默认路线的 1.9 倍，当前收益不足以覆盖计算成本。
4. 训练集 HPEC AUC 可达到 0.9 以上，但验证和测试没有同步提高，主要问题仍是表征泛化，而不是 prototype 是否足够分散。
5. MDD 默认入口继续使用原有 `mean_std + reliable_tp_ema + Busemann evidence` 路线；新增机制仅作为默认关闭的研究消融开关，不作为当前最佳配置。

## 6. 后续方向

下一轮不应继续增加 prototype 分离约束。更值得验证的是：在不增加四套 HPEC 的前提下，将因果角色统计作为共享 HPEC 的低维条件变量，或者用训练折内的站点/类别不变性约束改善模块 3 表征泛化。任何新方案仍需经过相同的完整五折测试后才能修改默认配置。
