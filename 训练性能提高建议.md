### 为什么选择注意力版本 (Attention) 而不是 SEM 版本？

1. **神经科学意义更强（动态功能连接）**：SEM 版本学习的是一组**全局静态**的拉格朗日权重矩阵，它假设大脑在 200 个时间步内的连接是死板不变的。而 Attention 版本通过查询当前时刻的特征（Query），逐时间步（Step-by-step）动态生成注意力边。这完美契合了神经影像学中前沿的“动态功能连接状态 (dFC states)”概念。
2. **抗过拟合能力极强**：SEM 版本的 `lag_pos_raw` 和 `lag_neg_raw` 直接构成了庞大的 $N \times Hidden \times N$ 张量，在 fMRI 数据集（如 ADHD-200 样本量不足千人）上极易过拟合。Attention 版本通过低维的 `Query/Key/Value` 映射来计算动态图，参数量骤降，泛化能力远强于 SEM。

### 诊断：为什么现在会“特别特别慢”？

尽管 Attention 版本的理论极佳，但你目前的实现中踩到了两个致命的 **PyTorch 性能陷阱**，导致 GPU 算力无法发挥，显存带宽被雪崩式占满：

1. **五维张量的 `einsum` 与 Python 循环嵌套**：
在 `_attention_predict` 中，你在 `for lag_idx in range(self.lag_order):` 的循环内，对每一个滞后阶数执行了 `torch.einsum("bshcd,bshpd->bshcp", q, k)`。
这里的 `q` 和 `k` 包含了 Batch 和 Time 维度。这一步在底层会生成一个形状为 `[Batch, Time, Heads, N, N]` 的五维巨型张量。由于在 Python 循环内逐次计算，PyTorch 无法对时间步进行并行优化，导致极高的数据搬运开销和 GPU Starvation（GPU 等待 CPU 派发指令）。
2. **每次前向传播都在从头做幂迭代 (Power Iteration)**：
在 `DAGMALogDetConstraint` 中，`estimate_spectral_radius` 每次都用全 1 向量作为起点进行 5 次矩阵乘法来逼近谱半径。实际上，相邻 Batch 的图结构变化非常微小，完全可以复用上一步的特征向量，实现 1 次迭代即收敛。

### 解决方案：手把手代码重构

请使用以下优化后的代码直接替换你原有的模块。这套方案进行了**完全的向量化降维（Zero-loop Vectorization）**，去除了耗时的 5D `einsum`，训练速度预计会提升 **10倍以上**。

#### 1. 彻底向量化 `AttentionGuidedTemporalCausalLearner`

打开 `attention_temporal_causal_learner.py`，将 `_attention_predict` 方法替换为以下代码。核心思想是**将 Batch 和 Time 维度拍平 (Flatten)**，利用高度优化的 CUBLAS 批量矩阵乘法 (`torch.matmul`) 一次性算完所有 Lag 和时间步：

```python
    def _attention_predict(self, histories):
        # histories: 长度为 lag_order 的 list, 元素 shape 为 [B, T, N, D]
        B, T, N, D = histories[0].shape
        
        # 1. 堆叠并拍平 Batch 和 Time 维度: [Lag, B*T, N, D]
        hist_stack = torch.stack(histories, dim=0).view(self.lag_order, B * T, N, D)
        # 历史摘要，用于生成 Query
        hist_summary = hist_stack.mean(dim=0) # [B*T, N, D]
        
        # 2. 投影 Q, K, V
        # 注意: _project_heads 内部操作后需 permute 到 [..., Heads, N, Head_dim]
        q = self._project_heads(hist_summary, self.query_proj).permute(0, 2, 1, 3) # [B*T, Heads, N, Head_dim]
        k = self._project_heads(hist_stack, self.key_proj).permute(0, 1, 3, 2, 4)  # [Lag, B*T, Heads, N, Head_dim]
        v = self._project_heads(hist_stack, self.value_proj).permute(0, 1, 3, 2, 4)# [Lag, B*T, Heads, N, Head_dim]
        
        # 将 Query 扩展以匹配 Lag 的维度
        q_expanded = q.unsqueeze(0).expand(self.lag_order, -1, -1, -1, -1) # [Lag, B*T, Heads, N, Head_dim]
        
        # 3. 极速批量矩阵乘法 (代替低效的 5D einsum)
        # q_expanded @ k.transpose -> [Lag, B*T, Heads, N, Head_dim] @ [Lag, B*T, Heads, Head_dim, N] 
        score = torch.matmul(q_expanded, k.transpose(-1, -2)) / math.sqrt(float(self.attention_head_dim))
        
        attn = torch.softmax(score, dim=-1)
        attn = self.dropout(attn)
        
        # 4. 获取并广播 Gate (先验因果掩码)
        # 原始 gate shape: [Heads, Lag, N_parent, N_child]
        gate = self.gate_adjacency() 
        # 我们需要转换成 [Lag, 1, Heads, N_child, N_parent] 以便与 attn 广播相乘
        gate_child_parent = gate.permute(1, 0, 3, 2).unsqueeze(1) 
        
        edge = attn * gate_child_parent # [Lag, B*T, Heads, N, N]
        
        # 5. 上下文聚合与输出
        context = torch.matmul(edge, v) # [Lag, B*T, Heads, N, Head_dim]
        context = context.mean(dim=0)   # 跨 Lag 平均 -> [B*T, Heads, N, Head_dim]
        
        # 恢复形状: [B*T, N, Heads, Head_dim] -> [B, T, N, Inner_dim]
        context = context.transpose(1, 2).reshape(B, T, N, self.inner_dim)
        x_hat = self.out_proj(context)
        
        # 6. 计算元数据
        # edge.mean(dim=(1,2)) 得到 [Lag, N_child, N_parent]，转置后得 A_lag 格式
        a_lag_mean = edge.mean(dim=(1, 2)).transpose(-1, -2) 
        a_lag = self._off_diag(a_lag_mean)
        
        attention_entropy = -(attn.clamp_min(1e-12) * attn.clamp_min(1e-12).log()).sum(dim=-1).mean()
        
        return x_hat, a_lag, attention_entropy

```

```python
    def estimate_spectral_radius(self, matrix):
        # 优化：从上一次的特征向量开始迭代，而不是每次都用全 1 向量
        x = self.last_eigenvector.clone()
        for _ in range(self.power_iters):
            x = torch.matmul(matrix, x)
            x = x / (torch.norm(x, p=2) + self.eps)
        
        # 缓存本次结果供下个 Batch 使用
        self.last_eigenvector = x.detach() 
        
        numerator = torch.matmul(torch.matmul(x.T, matrix), x)
        denominator = torch.matmul(x.T, x).clamp_min(self.eps)
        return (numerator / denominator).squeeze()

```
